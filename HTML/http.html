O Protocolo HTTP

O HTTP (Hypertext Transfer Protocol) é o protocolo base da Web. Ele atua como o mensageiro que dita as regras de como os navegadores e os servidores devem se comunicar para trocar arquivos (como HTML, imagens e vídeos).

A Dinâmica de Cliente e Servidor

A comunicação na Web funciona em um modelo de pergunta e resposta:

  • Cliente (Navegador): É o seu computador (Navegador). Ele inicia a conversa fazendo uma Requisição (pedido).
  • Servidor: É o Computador onde o site está hospedado. Ele processa o pedido e devolve uma Resposta.

Métodos: GET e POST

Quando o cliente faz uma requisição, ele usa "verbos" para dizer o que quer fazer:

  • GET: Usado para buscar informações. Quando você digita um site e aperta Enter, está fazendo um GET. Os dados enviados ficam visíveis na URL.
  • POST: Usado para enviar informações pesadas ou sensíveis ao servidor (como enviar um formulário de login ou senha). Os dados vão ocultos no corpo da requisição.

Códigos de Status HTTP

O servidor sempre responde com um número de 3 dígitos (status code) para avisar o que aconteceu com o pedido:

  • 200 (OK): Sucesso! A requisição foi recebida e a página foi encontrada.
  • 301 (Moved Permanently): O site ou página mudou de endereço definitivamente. O servidor redireciona o cliente para o novo link.
  • 400 (Bad Request): Erro do cliente. A requisição foi malformada ou o servidor não conseguiu entendê-la.
  • 403 (Forbidden): Proibido. A página existe, mas você não tem permissão para acessá-la.
  • 404 (Not Found): Não encontrado. A clássica página de erro quando o link não existe ou foi apagado.
  • 500 (Internal Server Error): Erro no servidor. O cliente fez tudo certo, mas o servidor "crashou" ao tentar processar.
